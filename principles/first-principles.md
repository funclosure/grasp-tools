# First Principles for Grasp Tools

## Working question

How do we build tools that extend mind without outsourcing grasp?

This document captures the first design criteria derived from early discussion around Bret Victor, John Vervaeke, and Andy Clark.

## Core distinction

### Participatory extension
A tool extends cognition by recruiting more of the human being into the act of knowing.
It leaves the person more capable, more oriented, and more able to think.

### Substitutive outsourcing
A tool delivers useful outputs while bypassing the growth of understanding, salience, skill, or participation.
It may increase convenience while leaving the person thinner or more dependent.

## Design criteria

### 1. The Grasp Test
Does the tool increase actual understanding, or only deliver a useful output?

A good tool should leave the user with:
- better orientation
- clearer distinctions
- stronger mental models
- improved ability to act again later without the tool carrying all the intelligence

### 2. The Participation Test
Does the tool preserve and cultivate active involvement?

A humane cognitive tool should invite:
- interpretation
- manipulation
- comparison
- exploration
- judgment

The user should remain a participant, not merely a requester or consumer.

### 3. The Multi-Knowing Test
Which kinds of knowing does the tool support?

A narrow tool may optimize propositional delivery.
A richer tool may help cultivate:
- propositional knowing
- procedural knowing
- perspectival knowing
- participatory knowing

A humane tool should support more than fact transfer alone.

### 4. The Salience Test
Does the tool help the user notice what matters?

Good cognitive tools do more than answer explicit questions.
They can help surface:
- patterns
- tensions
- anomalies
- tradeoffs
- better questions

This matters because relevance realization is central to grasp.

### 5. The Agency Test
Does the tool increase the user's independent capacity over time?

A humane extension should grow:
- discernment
- confidence
- self-direction
- ability to think and act without constant dependence on the tool

### 6. The Transparency Test
Can the user see how the scaffold is shaping cognition?

A strong tool should make visible, where possible:
- assumptions
- sources
- uncertainty
- provenance
- transformations
- what is scaffolded versus what is inferred

### 7. The Transfer Test
Does learning with the tool carry beyond the tool?

A real grasp-tool should create transferable gains:
- better concepts
- better questions
- better habits of noticing
- better inquiry practices

If the user only functions well inside the tool, the scaffold may be enclosing rather than extending.

### 8. The Embodiment Test
Does the tool recruit richer human capacities where appropriate?

This does not mean physicality for its own sake.
It means asking whether the tool could better recruit:
- spatial reasoning
- tactile interaction
- temporal rhythm
- social interaction
- movement
- environmental coupling

The aim is a richer ecology of cognition, not a fetish for bodily form.

### 9. The Transformation Test
Does the tool only inform, or can it transform the user?

The highest bar for a humane cognitive tool is not mere efficiency.
It is whether the tool can contribute to:
- seeing differently
- caring differently
- acting differently
- becoming different

This is where participatory knowing becomes central.

## Compact definition

A grasp-tool is a cognitive scaffold that leaves the human more capable, not more replaceable.

## Open direction

These principles are early and incomplete.
They should be tested against real tools, real practices, and eventually real prototypes.
